로그인 후 통신이 필요한 경우, 인증과 로그인 관련 요청은 /api/auth(예시) 에서 처리, 즉 모든 URL을 auth로 할 필요는 없음

앞으로 다른 API를 연결할 때도 같은 패턴을 사용하면 됩니다. 아래는 주요 단계를 요약한 것입니다:

백엔드 API 설계: Spring Boot에서 REST API를 설계합니다. 보통 @PostMapping, @GetMapping 등을 사용하여 HTTP 요청을 처리합니다.
프론트엔드에서 데이터 가져오기: HTML에서 입력 필드를 통해 사용자의 데이터를 가져옵니다.
데이터 전송: fetch 또는 Axios 같은 HTTP 요청 라이브러리를 사용하여 API에 데이터를 전송합니다.
응답 처리: 서버의 응답에 따라 성공 또는 실패 시 적절한 피드백을 사용자에게 제공합니다.
이 패턴을 여러 상황에 적용할 수 있으며, 다른 기능을 추가할 때도 비슷한 방식으로 API 연동을 구현하면 됩니다.
